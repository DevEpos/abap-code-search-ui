ABAP Code Search +

The ABAP Code Search + is a tool which offers the possibility to search in abap source code for a given string, similar to the ABAP Source Code Search. The advantages of the ABAP Code Search+ are:

  • works also for R/3 environments
  • allows a more detailed scope definition with filtering by object type, application component, etc...
  • supports more specific search patterns with regex and multiline search
    • It is integrated in the eclipse Search dialog, which can be called via STRG + H or the eclipse menu Search âž  Search...

    The User Interface

    The Code Search Dialog Page is structured into the following controls

    1. A Text Area to enter the search pattern
    2. 3 Checkboxes to configure the search pattern regarding case sensitivity, if regular expressions are used and if the search string should be applied as a single pattern ( relevant for multiline strings)
    3. A text input control to enter object name to restrict the scope
    4. A text input control to enter the filters for the scope. The control supports content assist
    5. 3 Checkboxes to configure the search scope: if comment lines should be ignored, if it is a multiline search and if the matches should include all lines from the search pattern
    6. A link to the preferences, where some more project specific configuration options can de done
    7. A Scale to control the maximum number of search objects that should be scanned
    8. A text input control to enter the target ABAP Project. The control supports content assist

    Configuring the search pattern

    You have 3 option to configure the search pattern

    1. Ignore Case: setting this check box will result in applying the search pattern in upper case
    2. Regular Expressions: by setting this checkbox you specify that the entered pattern uses regular expression ( depending on the system you can even use PCRE see project specific settings)
    3. Single Pattern mode: To-do

      Additional search settings

      Use logical AND for options of search filters

      Normally the options of a search filter are interpreted with a logical OR . So if you have the following query: field: product,plant , the search would return all objects that have a field named product or a field named plant

      But sometimes you may want to find only the objects that have all the specified fields. In this case you have to activate the setting Use "AND" instead of "OR" during search. You can do this by setting the checkbox in the search dialog (see Figure 1)

      Regardless of it's activation state this setting will only be used for search filters where it makes sense, i.e. where multiple values are at all possible.

      The Search result View

      Working with the Search Results

      Once a search returns some results they are displayed in the Search View. For each found object there are several options available

      • Opening the object in the corresponding eclipse editor (if one is registered for the type) by double clicking on it
      • Open one or several selected objects via the context menu item Open

      If the tree node represents a CDS View, a Database Table or a Database View There are also the following options available via the context menu

      • Open the selected objects in the Data Preview of ADT
      • Open the selected objects in the SAP GUI Transaction DB Browser
        Note: The DB Browser feature needs to be installed in the ABAP System for this to work

      Sub Nodes of a Result Object

      Every Result Object can be expanded to at least one level. Upon expansion of one of these ADT objects, the sub nodes of the object will be loaded. During the loading of the object's sub nodes you will see a sub node called Loading Contents....

      Once the sub nodes are available they will be displayed under the ADT object
