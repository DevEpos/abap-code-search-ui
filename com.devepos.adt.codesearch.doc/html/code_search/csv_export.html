File Export for Code Search Results

To export the visible search results to a delimited text file you can use the export function which is available from the view menu of the search result page.

This action opens up a dialog with some configuration options on how the results shall be saved to a text file

First you have to select a file via the Browse button which opens up a system file dialog. If the chosen file already exists, you will get a warning message in the dialog that it will get overwriten upon completing the dialog.

Additionally you can choose between Comma, Semicolon and Tab as a delimiter.

Emphasize your matches

It may also be helpful to activate the option Use Marker sequence for Match. This will use the specified Marker Sequence to wrap the found search pattern in the code snippet (e.g. " %%TODO%%: clean up algorithm for the pattern todo ).

Columns in the Export

The following columns are currently written to the text file:

Column Description
Package Hierarchy Path Holds the full package hierarchy of the object separated by ">"
Source Package Contains the actual package of the repository object
Owner The id of the responsible person
Name The name of the main repository object (e.g. of a global class)
Object Type The description of the main repository object type
Sub-Object Name The name of the sub object, if available (e.g. Method Name or Include)
Sub-Object Type The type of the sub object (e.g. Method, Class Include, etc.)
Line Number The first line of the match
Source Code Extract The first source code line where the match was found
Link to Match ADT Link to the match in the repository object
